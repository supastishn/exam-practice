Setting Up Google AI Studio (OpenAI Compatibility)

Google AI Studio provides access to Gemini models, and you can use its API with an OpenAI-compatible endpoint. This allows you to use Gemini models with this application.

Privacy Warning: Google may use data submitted to Gemini models via the API to improve their products. Please review Google's Terms of Service and Gemini API Terms for more information. Do not send sensitive or confidential information.

Step 1: Access Google AI Studio & Get an API Key

  1. Go to Google AI Studio.
  2. Sign in with your Google account.
  3. Once in AI Studio, look for an option like "Get API key" (often found in the left sidebar or under a "Get started" section).
  4. Click on "Create API key in new project" or select an existing project.
  5. Copy the generated API key. Treat this key like a password and keep it secure.

Step 2: Configure in This Application

  1. Go to the API Key Settings page in this application.
  2. Paste the copied Google AI Studio API key into the "OpenAI API Key" field.
  3. In the "OpenAI Base URL (optional)" field, enter: https://generativelanguage.googleapis.com/v1beta/openai
  4. For the "Default Model Name (optional)" field, you can use one of the following (or other models compatible with the API key and base URL):
    • gemini-1.5-flash-preview-0514 (Supports text and images. Note: User mentioned gemini-2.5-flash-preview-04-17 with 500 requests/day; gemini-1.5-flash-preview-0514 is a recent performant model. Please verify exact model name and limits in your Google AI Studio console.)
    • gemini-2.5-flash-preview-05-20 (A newer preview model. Please verify exact model name, capabilities, and limits in your Google AI Studio console.)
    • gemini-1.0-pro (Supports text and images. Note: User mentioned gemini-2.0-flash with 1500 requests/day, potentially less performant than 1.5 Flash. gemini-1.0-pro is a widely available model. Please verify exact model name and limits.)
    • Check the official Gemini models documentation for the most current list of compatible model names for the API. The model name you enter here must be exactly as expected by the generativelanguage.googleapis.com API. For example, it might be models/gemini-1.5-flash-latest.
  5. Click "Save Credentials".
  6. You can use the "Test API" button to verify your setup. Ensure the model name used for testing is compatible.

Important Notes:

  • Model Naming: The exact model names (e.g., gemini-pro, models/gemini-1.5-flash-latest) required by the API might differ from display names in AI Studio. Refer to Google's API documentation for the correct identifiers to use in the "Default Model Name" field.
  • Capabilities: Gemini models often support multimodal inputs (text and images). Ensure the model you select supports the type of input you intend to use.
  • Rate Limits: Free tiers of the Gemini API typically have rate limits (e.g., requests per minute or per day). The user mentioned specific limits: 500 requests/day for one model and 1500 requests/day for another. Check your Google AI Studio console or documentation for the limits applicable to your key and models.
  • Security: Your API key is stored encrypted in your browser's local storage. Clear it if you are on a shared computer.

That's it! You should now be able to use the application with Google's Gemini models via their OpenAI-compatible endpoint.
